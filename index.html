Since I don't like other people's code …

Please misunderstand the heading. Other people's code is great, but I prefer to assemble my own HTML and CSS boilerplate. There are great projects out there, like the HTML5 Boilerplate, Skeleton, and LESS Framework; but I really can't just drop in their markup and CSS into my projects without a thorough investigation and reverse-engineering session first. Therefore I've created this personal toolbox of mine, which I always use when I start off a new project.

What is this?

This 'Compass Kit' of mine is a fluid, responsive, boilerplate in HTML and CSS, built with Compass and Sass. It evolved from my personal work and laziness – who has time to completely rebuild the boilerplate HTML and CSS for every project? Summary: this is my personal toolbox, and the code may change due to my mood, industry advice, and by plain obsessive compulsive refactoring. I usually drop this package into newly created projects, adjust and set some site specific variables, perhaps remove stuff I don't need, and start building.

Thanks to my obsessive eagerness to collect all the web development world's best practices, this kit includes up-to-date cutting edge stuff, as well as responsible fallbacks for older browsers. Progressive enhancements – oh yeah.

This kit includes as few CSS classes and static values as possible. It's not up to me to decide how you organize your CSS. I personally hate it when a CSS framework dictates everything, and I end up deleting tens of lines of code just because I don't need it. This kit doesn't get in your way, and even includes a customized version of Normalize.css CSS reset instead of a regular shotgun reset. Thanks to Normalize.css, we get a nice cross-browser and consistent markup environment instead of hard reseting all elements.

Compass & Sass

There's one thing different with this package though: it's built with Sass and Compass. You should start use those technologies right now. It's like CSS4, but available for us to use today. Please study the Compass reference for tons of ways to use the tool. I'm sure you'll find a couple of nifty solution in this package's code. For a quick walkthrough, check out the readme on GitHub.

What's in the box?

  • Great typography setup based on a modular, golden, scale (reference table)
  • Handy Sass mixins
  • Organized file structure (see below)
  • Helper classes, and more.

Modularity

I hate messy CSS files. However, Sass provides the @import construct, where we can import a CSS or Sass file into another Sass file, without using more HTTP requests. It's great. Take a look at the directory structure in the GitHub repo: it really is a "each file in its own place" philosophy:

See what's going on to the left? If you do, I know I've done this right. Each file in its own place. Feel free to come up with your own organization. I've found this structure quite neat – separating the CSS code into modules is sooo good. All of the files in the sass folder compiles into a single master.css file. Awesome.

The goal of a file strucure like this is when you're looking for a specific selector, you should just know where that is located. It shouldn't have to take you more than a couple of seconds to look that up.

Quick & Dirty

What follows is all commonly used HTML elements (including the ones in defined in HTML5) listed, for rapid testing and experimenting. I've included the Javascript tool Hashgrid which lets you define grid overlays to make sure everything line up. Try it out by pressing and holding 'G' on your keyboard. This 12 column grid has a 24 px baseline and is 992px wide. This makes every column 68px wide and the gutters 16px Measurements may be adjusted in the files _base.scss and library/_hashgrid.scss – it's completely up to you. Thanks to the variables in Sass, there are no hardcoded widths in my boilerplate.

Outro

I really don't have the time to build one of these fancy promo sites for this boilerplate package. As I mentioned: this is primarly for myself. If you want it, it's in a GitHub repo (you can clone or download over there). The included Readme should provide enough intel to get you off the ground.

There are no crazy CSS class names to keep in mind, just a couple of helper classes. The best way to learn is actually to study the code – not reading some abstract readme – so dive in! Study the import scheme, the variables, the custom mixins. You'll like it.

His room, a proper human room although a little too small, lay peacefully between its four familiar walls. A collection of textile samples lay spread out on the table - Samsa was a travelling salesman - and above it there hung a picture that he had recently cut out of an illustrated magazine and housed in a nice, gilded frame. It showed a lady fitted out with a fur hat and fur boa who sat upright, raising a heavy fur muff that covered the whole of her lower arm towards the viewer.
